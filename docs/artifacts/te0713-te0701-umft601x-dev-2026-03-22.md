### Timing fix
- Registered `ft601_txe` and `ft601_rxf` inputs in `usb_data_interface.v`
- IOB TRUE constraints pack input registers into ILOGIC sites
- Eliminates all top-10 critical paths from unregistered input pads

### Self-test integration
- `fpga_self_test.v` instantiated in dev wrapper (was hardwired in v0.1.8)
- Opcode 0x30 triggers 5-subsystem self-test FSM
- Opcode 0x31 reads latched results via status mechanism
- Tests 0-3 (BRAM, CIC, FFT, MTI arithmetic) run on real hardware
- Test 4 (ADC capture) will timeout on dev board (ADC inputs tied to 0)

## What This Bitstream Does

- Instantiates `usb_data_interface.v` (full FT601 USB data path) and
  `fpga_self_test.v` (5-subsystem self-test) via dev wrapper
- Generates synthetic test data: range profile packets (counter XOR pattern),
  optional Doppler and CFAR packets, controlled by host commands
- Responds to USB host commands (stream control 0x04, self-test 0x30/0x31, status 0xFF)
- Drives `ft601_gpio0` with a ~6 Hz heartbeat (counter bit 24)
- Drives `ft601_chip_reset_n` after power-on reset (32k clock cycles)
- `ft601_wakeup_n` held high (inactive)

## Hardware Setup

1. TE0713-03 mounted on TE0701-06 carrier
2. UMFT601X-B plugged into TE0701 J10 (FMC LPC connector)
3. TE0701 VIOTB = 3.3V (jumper configuration)
4. UMFT601X-B jumpers: JP1=open, JP2=2-3, JP3=open, JP6=short (3.3V),
   JP4=2-3, JP5=2-3 (245 Sync FIFO mode)
5. USB 3.0 Micro-B cable from UMFT601X-B to host PC
6. Trenz programming USB for JTAG

## Verification Steps

1. After programming, check `ft601_gpio0` toggles (~6 Hz) — confirms FPGA running
2. On host PC, FT601 should enumerate as USB 3.0 device (VID 0x0403, PID 0x601F)
3. Run host-side smoke test:
   ```
   cd 9_Firmware/9_3_GUI
   python smoke_test.py --live
   ```
   Expected: Tests 0-3 PASS, Test 4 FAIL (no ADC on dev board)
4. Use dashboard for live data:
   ```
   python radar_dashboard.py --live
   ```
